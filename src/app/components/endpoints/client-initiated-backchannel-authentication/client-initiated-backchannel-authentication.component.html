Required parameters

  • claims - the claims which will be accessible by userinfo endpoint (if openid authorization detail type is requested). The sub claim must be requested (or already granted if grant_id is specified)
  • client_notification_token - required if client is registered to use Ping mode
  • id_token_hint or login_hint - one of these must be present. The login_hint value is by default treated as subject - unique user identifier

Optional parameters

  • authorization_details - see Rich Authorization Requests
  • binding_message
  • user_code - by default it's optional, but can be required by specific user. See Implementing necessary services
  • grant_id, grant_management_action - see Grant Management RFC It can be required, but it depends on provider configuration
  • dpop_pkh - see Demonstrating Proof Of Possession
