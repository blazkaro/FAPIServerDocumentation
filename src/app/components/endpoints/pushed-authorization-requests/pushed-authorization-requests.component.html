Required parameters

  • client_id, client_assertion_type, client_assertion - see Client Authentication
  • claims - the claims which will be accessible by userinfo endpoint (if openid authorization detail type is requested). The sub claim must be requested (or already granted if grant_id is specified)
  • redirect_uri
  • state - by default, the length must be between 43 and 384
  • nonce - by default, the length must be between 43 and 384
  • code_challenge_method - it must be S256
  • code_challenge - it must be Base64-Url encoded string, by default with length between 43 and 128

Optional parameters

  • authorization_details - see Rich Authorization Requests
  • grant_id, grant_management_action - see Grant Management RFC It can be required, but it depends on provider configuration
  • dpop_pkh - see Demonstrating Proof Of Possession
  • prompt - supported values are: login. To request fresh grant use grant_management_action
  • max_age
