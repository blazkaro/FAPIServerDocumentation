Every requset to this endpoint must be authenticated. See Client Authentication

Required parameters

  • token - if it starts with v4.paseto. it will be processed as access token. Otherwise, the provider will assume that's refresh token

This endpoint provides response signature. The header X-Body-Security-Token is added to response and contains v4 public PASETO signed by authorization server. You can verify signature with the same public key which you use to verify id_token. This PASETO contains hash claim which is Base64-Url encoded hash of response body in JSON format (without null values).
